Problem Statement

Traditional ML workflows require manual preprocessing, model selection, and evaluation which demand expertise and time. There is a need for an intelligent system that automatically constructs optimal ML pipelines for structured datasets without requiring domain knowledge from the user. A²ML addresses this by bridging the gap between raw data and deployable models through a policy-driven autonomous engine.

🎯 Objective

Design and implement a dataset-aware autonomous ML pipeline that:

System Module Specification

Module File Responsibility
Data Input Engine data_input.py Loads CSV, detects column types, identifies target variable, applies problem type detection rules.
Dataset Analysis Engine meta_learning.py Computes: number of samples, feature types, missing ratio, skewness, class imbalance, and complexity score.
Adaptive Preprocessing preprocessing.py Decision-policy preprocessing: KNN imputation if missing >20%, encoding for categoricals, normalization for skewed distributions.
Feature Engineering feature_opt.py Strategy selection via dataset shape: polynomial interactions (≤10 features), PCA (features > rows), or mutual information.
Multi-Model Training model_training.py Instantiates 13 algorithms across regression, classification, and clustering. XGBoost loaded conditionally.
Hyperparameter Optimization hyperparameter.py GridSearchCV with predefined grids per model. Falls back to default fit if no grid defined.
Model Evaluation Engine evaluation.py Benchmarks all trained models. Auto-selects best using RMSE (regression), F1 (classification), or Silhouette (clustering).
Explainable AI Engine explainability.py SHAP feature importance (TreeExplainer / LinearExplainer / KernelExplainer) + Partial Dependence Plots for top feature.
Knowledge Memory System knowledge_memory.py Stores metadata vectors per run. Recommends models via Euclidean distance on [rows, features, missing ratio, complexity].
Pipeline Orchestrator pipeline.py Chains all engines in correct sequence, manages data flow, returns structured results dictionary.

Dataset Complexity Score

📐 Official Formula

complexity = 0.4 × (num_features / num_samples)
+ 0.3 × (missing_ratio)
+ 0.3 × (skewness_score)
Score Range Label Meaning
< 0.15 Low Dataset is clean, well-sized, and normally distributed
0.15 – 0.35 Medium Moderate dimensionality or missing values present
≥ 0.35 High Highly dimensional, skewed, or heavily missing data

Skewness score is normalised to [0, 1] using cap at mean absolute skewness = 10.

Adaptive Preprocessing & Theoretical Justification

🔧 Missing Value Imputation

if missing_ratio > 20% → KNN Imputation
else → Simple mean/mode imputation

Theoretical Basis: KNN preserves local structure in high-missingness regimes, whereas mean imputation preserves global variance but distorts local density.

🔤 Categorical Encoding

if cardinality ≤ 10 → One-Hot Encoding
else → Label Encoding

Theoretical Basis: One-Hot avoids ordinal assumptions for low-cardinality features. Label encoding prevents dimensionality explosion for high-cardinality nominals.

📏 Feature Scaling

StandardScaler (Z-score normalization)
applied to all numeric features

Theoretical Basis: Centering and scaling ensures gradients descend uniformly for optimization-based models (SVM, Ridge) and avoids distance dominance in KNN.

🔬 Feature Engineering

≤ 10 features → polynomial interactions
features > rows → PCA
else → mutual information selection

Theoretical Basis: PCA prevents overfitting in p > n regimes; interactions capture non-linear feature couplings in low-dimensional spaces.

Knowledge Memory & Advanced Similarity Search

🧠 Multi-Factor Similarity

Similarity is no longer just on count, but on informational distribution. Metadata vector:

meta_vector = [rows, features, missing_ratio, complexity, correlation, entropy]

Correlation: Mean absolute Pearson correlation between numeric features and target.
Entropy: Mean Shannon Entropy of categorical features (proxy for information richness).
The system uses Euclidean distance on normalised vectors to retrieve the best historical model.

Research Contributions

1. Dataset-Aware Preprocessing Framework

Preprocessing decisions are driven by statistical properties of the input dataset — not hardcoded rules — enabling generalisation across diverse data distributions.

2. Automated Model Selection Strategy

Multi-model training with automated selection based on problem-type-specific metrics (RMSE / F1 / Silhouette), eliminating subjective human judgment from model choice.

3. Adaptive Feature Engineering Pipeline

Strategy selection (PCA / MI / Interactions) is determined programmatically from dataset shape characteristics without requiring user expertise.

4. Euclidean-Distance Knowledge Memory

The system learns from previous executions, using Euclidean distance on normalised metadata vectors to recommend models for future similar datasets.

5. Explainable ML Pipeline

SHAP values and Partial Dependence Plots provide dual-layer model transparency — both overall feature magnitude and directional effect on predictions.

6. Complexity Score Formula

A defined numeric complexity metric (feature density + missing ratio + skewness) enables reproducible dataset characterisation for future research.

Engineering Rigor & Production Realism

📂 Configuration System

Central config.yaml controls all pipeline hyperparameters, test ratios, and complexity weights without code changes.

💾 Model Persistence

Automatic joblib dumping of the Auto-Selected Best Model to logs/best_model.pkl for production deployment.

📝 Detailed Logging

Multi-level logging to logs/a2ml_YYYYMMDD.log capturing precision execution timestamps and internal engine state.

⏳ Overfitting Control

Composite selection score penalizes models with large train-test gaps: final = score - (0.5 * overfit) - (0.01 * time).

Error Analysis & Time Complexity

🔍 Error Handling & Fallbacks

  • Small Datasets: Automatically disables hyperparameter optimization if rows < 15 to prevent CV fold failure.
  • Missing Values: Policy fallbacks from KNN to Simple Mean if feature variance is too low for computation.
  • XAI Failure: SHAP KernelExplainer sampled subsetting to prevent memory overflow on wide datasets.

⏱️ Time Complexity Analysis

Overall training complexity is O(M × (N_train + CV_loops)) where M is model count. Inference is O(1) for most selectors, keeping it deployment-ready.

System Limitations

  • Supports tabular (CSV) data only — no image, audio, or text support
  • Meta-learning capability is limited to Euclidean-distance-based recommendation (no deep meta-features)
  • Computational cost is high for very large datasets (>100k rows)
  • Result quality is dependent on the quality of the input dataset
  • XGBoost on macOS requires installation of the libomp system library
